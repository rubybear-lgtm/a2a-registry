## Design Context

### Users
Mixed audience — **developers/builders** publishing and integrating AI agents, and **enterprises/ops teams** evaluating agents for deployment. Both audiences are technically literate and value information density over decoration. They need to scan, compare, and assess agents quickly.

### Brand Personality
**Bold, futuristic, innovative.** The registry is infrastructure for the agentic web — it should feel like a serious, forward-looking platform, not a side project. Think Linear, Railway, Vercel: confident typography, crisp monochrome, design that earns trust through precision.

Three words: **precise · authoritative · forward-looking**

Emotional goal: Users should feel they're looking at the definitive, reliable source of truth for AI agents — not a demo or prototype.

### Aesthetic Direction
- **Minimal & data-dense.** Content-forward layout with tight spacing. Tables, badges, and monospace for technical values (versions, protocol bindings, URLs). Let the data breathe — but not too much.
- **Neutral color palette** (keep existing near-black/white + dark mode tokens). No decorative color. If accent is ever needed, use it sparingly and deliberately.
- **Bold typographic hierarchy.** Large, confident headings to anchor sections. Instrument Sans already in use — lean into its weight range.
- References: npm registry, crates.io, Linear's issue list view, Railway's dashboard.
- Anti-references: Colorful card grids, playful illustration, marketing-heavy hero sections.

### Design Principles
1. **Signal over decoration** — Every visual element must earn its place by conveying information. No ornamental color, gradient, or illustration.
2. **Data is the interface** — Agent metadata (version, capabilities, skills, protocol) should be scannable at a glance. Use monospace, badges, and structured layouts to make structure visible.
3. **Confidence through restraint** — The palette is neutral, the type is precise, the spacing is tight. Bold means typographic boldness, not visual loudness.
4. **Dark mode as a first-class experience** — The registry will be used in terminals-adjacent contexts. Dark mode should look purposeful, not just inverted.
5. **Progressive disclosure** — Lists show essential fields; detail pages expand. Don't overwhelm — let users drill down.
